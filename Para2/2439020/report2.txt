回答1.2 : イベントハンドラを登録したオブジェクトは次のとおり
scene




イベントハンドラを上のように登録した理由は以下のとおり
キーボード入力イベントをシーン全体で受け取れるからだ。もし各ボータンに陰ベントハンドラを登録した場合は、
それぞれのボタンに対して処理を書く必要があり、コード量が増えて管理しづらくなるからだ。
一方、sceneに登録すると、どのキーが押されても一つのイベントハンドラで処理できるため実装しやすい。
また、ユーザがキーボード数字や演算子を入力した場合でも、どのボタンにふぉかすがあるかを意識する必要なく、シーン全体で入力を受け付ける。そのため、様々な
キー入力に柔軟に対応でき、拡張しやすい。




回答4 : package para.graphic.parserの動作の説明
String data =
"shape 0 Circle 10 10 40 Attribute Color 100 40 60
Fill true\n"+
"target draw\n"+
"target flush\n";
MainParser mp = new MainParser(target,data);
mp.parse(data);

MainParser.parse() が data 全体を受け取り、Scanner を用いてトークンを先頭から順番に最後まで読み取る。
最初に読み込まれるトークンは shape である。
MainParser のコンストラクタでは次のように HashMap に各コマンドと対応する Parser が登録されている。

map.put("shape", new ShapeManagerParser(shapemanager));
map.put("target", new TargetParser(target, shapemanager));
map.put("reset", new ResetParser());

shape を読み取ると、mp = map.get(s.next());が実行される。
map.get("shape") によって ShapeManagerParser のインスタンスが取得され、 mp.parse(s)が実行される。


ShapeManageParserの parse関数の中身：
  public void parse(Scanner s) {
    int id = s.nextInt();
    ShapeParser sp = map.get(s.next());
    shapemanager.put(sp.parse(s, id));
  }

次のトークン 0 を読み取り、id に代入し、次のトークンを読み取ると Circle である。
ShapeManagerParser のコンストラクタではmap.put("Circle", new CircleParser());だからCircleParser.paser()が実行される。


CircleParserのparse関数の中身：
  @Override
  public Circle parse(Scanner s, int id){
    int x = s.nextInt();
    int y = s.nextInt();
    int r = s.nextInt();
    Circle ret;
    Attribute attr=null;
    if(s.hasNext("Attribute")){
      attr = AttributeParser.parse(s);
    }
    ret = new Circle(id,x,y,r,attr);
    return ret;
  }

x = 10, y = 10, r = 40 を順番に読み取る。次のトークンが Attribute なのでAttributeParser.parse(s);graphic呼び出される。

Attribute の次のトークンは Color なのでswitch 文によって Color の処理が選択される。
100, 40,60 をそれぞれ r, g, bとして読み取る。さらに Fill true を読み取り、fill = trueとなる。

CircleParser に戻り、読み取ったid, x, y,r,Attributeを用いてnew Circle(id, x, y, r, attr);を実行する。
作成した Circle オブジェクトを ShapeManagerParser に返し、ShapeManager への追加
ShapeManagerParser に戻り　shapemanager.put(sp.parse(s, id));が実行される。
返された Circle オブジェクトが ShapeManager に追加される。

ShapeManagerParser.parse() が終了すると、処理は再び MainParser.parse() に戻る。
次に読み込まれるトークンは target であり、map.get("target") により TargetParser が取得される。
TargetParser.parse(s);が実行され、次のトークンは draw なので、DrawParser.parse() が呼び出される。
その中でtarget.draw(shapemanager);が実行され,ShapeManager に保存されているすべての図形に対して draw() が呼ばれ、画面へ描画される。

MainParser.parse() に戻り、次のトークンも target なので、再び TargetParser.parse() が呼び出される。
今度は次のトークンが flush なので、target.flush();が実行される。
バッファの内容が画面へ反映される。

以上で data 内のすべてのトークンの解析が終了する。


if-else文を並べた実装方法と比べた拡張性に関する利点はつぎのとおり
if-elseで実装する例：
while(s.hasNext()){

    String command = s.next();

    if(command.equals("shape")){
        shapeParser.parse(s);
    }
    else if(command.equals("target")){
        targetParser.parse(s);
    }
    else if(command.equals("reset")){
        resetParser.parse(s);
    }

}
今回実装した例：
mp = map.get(command);
mp.parse(s);

if-else 文では、コマンドが増えるほど条件分岐が長くなり、コードが読みにくくなる。
HashMap を利用すると、新しいコマンドを追加する場合はmap.put("新しいコマンド", new ○○Parser());を追加するだけでよい。
MainParser.parse() の処理自体を変更する必要がないため、保守性・拡張性が高い。
各 Parser がそれぞれの処理だけを担当するため、役割が明確になり、コード全体の管理や理解がしやすくなる。



回答5 : para.graphic.parserパッケージの各クラスのアクセス修飾を次のようにした
AttributeParser       public -> 無修飾
CircleParser          public -> 無修飾　
ImageParser           public -> 無修飾　
MainParser            public -> public
MetaParser            public -> 無修飾
RectangleParser       public -> 無修飾
ShapeManagerParser    public -> 無修飾
ShapeParser           public -> 無修飾
TargetParser          public -> 無修飾

上記のようにアクセス修飾をした理由は次のとおり
パッケージ外から参照する必要があるクラスには public を付ける。MainParserは mainのファイルによく呼ばれるのでそれをpublicにする必要がある。
残りのクラスは、例えばAttributeParser や CircleParser などは MainParser からのみ利用される。
パッケージ外から直接利用されることはないため、無修飾（package-private）にした。
privateとprotectedを使ったらアクセスできなくなるため、今回は使用しなかった。






回答6 : para.graphic.parser.ImageParserの仕組みの説明

ImageParser には二つのメソッドがあり、parse()とscanImage()

1. parse()関数
まず、データからx,yの二つのint型変数を読み取り、BufferedImage class imgを作る。
画像データの読み込みに失敗した場合は、10×10ピクセルの空の BufferedImage を生成して代わりに使用する。

データにはattributeがあったらそれを読み取り、すべての変数がそろったらImage classのインスタンスとしてreturnする。

2. scanImage()
このメソッドはBase64形式で保存された画像データを読み込み、BufferedImage変換する処理で行う。
まず、Scanner と Pattern を利用して Base64 文字列を一つずつ読み取る。
Pattern は Base64 で使用される文字のみを読み取るために用意されている。
while(!(str.length()==1 && str.hashCode()==codeand))で読み取った文字列は "&" が現れるまで while 文で繰り返し処理され、System.arraycopy() を用いて bisdata 配列へ順番にコピーされる。
ここで "&" は画像データの終了を表す区切り文字として利用されている。

画像データをすべて読み込んだ後、ByteArrayInputStream の reset() メソッドによって読み込み位置を先頭に戻す。
Base64.Decoder の wrap() を利用して Base64 文字列を元のバイト列へデコードし、
そのデータを ImageIO.read() に渡して BufferedImage を生成する。
最後に、生成した BufferedImage を parse() メソッドへ返し、Image オブジェクトの生成に利用する
