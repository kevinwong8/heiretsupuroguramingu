回答2.2 : 説明は次のとおり
各ソースコード内のpackageはpackage-info.javaのファイルに保存されている。
例えば Paint.javaの書いてある package para.paint と指定すると、クラスはpara.paint の package に属する。
javacの-d binオプションを用いると、コンパイラはpackage名に対応するdirectory構造(para/paint)をbinの下に自動生成し、bin/para/paint/Paint.classにclassファイルを出力する。
また、java commandでは-cp binを指定してbinをクラスパスの基準directoryとし java -cp bin para.paint.Paintのように「package name + classname」を指定して実行する.

回答3.4 : 機能拡張を次のように行った

追加機能その1
　具体的な実装の説明
画像の90度回転をさせるボタンの追加を実装した。
画像を90度回転させる機能として、rotateL（左回転）ボタンとrotateR（右回転）ボタンを追加した。
それぞれのボタンにsetOnAction()を設定し、押下時にsetRotate()を用いてCanvasを回転させるようにした。
回転角度は、(canvas.getRotate() ± 90) % 360
によって計算した。現在の回転角度に90度を加算または減算し、360で剰余を取ることで、
回転角度が過度に大きくなったり小さくなったりすることを防いだ。


追加機能その2
　具体的な実装の説明
一つ前の描画操作の取り消し機能の追加を実装した。
実装にはStackとWritableImageを利用した。
マウスが押された時点でCanvasの状態をWritableImageとして保存し、
その画像をStackにpush()して履歴として記録した。
Undoボタンが押された場合には、Stackから最新の画像をpop()で取り出し、
clearRect()でCanvasを消去した後にdrawImage()を用いて保存されていた画像を再描画した。
Stackは後入れ先出し（LIFO: Last In First Out）のデータ構造であるため、
直前の描画状態から順に取り消しを行うことができる。


回答4.1 :
　実装に工夫した点があれば説明
電卓のボタンを4行に配置するため、HBoxを4個（Hb0～Hb3）作成し、それぞれを1行として使用した。
ボタンの生成にはbuttons[]とbuttonname[]を利用した。buttonname[]を参照しながらbuttons[]を走査し、
各要素に対応するボタンを動的に生成した。これにより、同じ処理を繰り返し記述する必要がなくなり、ソースコードを簡潔に記述できた。
各ボタンにはsetOnAction()を設定し、数字や演算子のボタンが押された場合には、その文字をbuffに追加した。
その後、buffをStringに変換してinputに表示し、現在の入力内容を確認できるようにした。
一方、calculateボタンとdeleteボタンについては通常のボタンとは異なる処理が必要であるため、
それぞれ個別にsetOnAction()を実装した。
calculateボタンが押された場合にはreverse_polish()を呼び出して計算を行い、計算終了後にinputを空にした。
deleteボタンが押された場合にはbuffの最後の文字を削除し、その内容を再びinputに表示するようにした。
最後に、HBoxとVBoxを組み合わせて各ボタンを配置し、見やすいレイアウトとなるように整理した。

回答4.2 :
  (イベントハンドラの登録の実装をどう行ったかを説明
  工夫した点があれば、その説明)

reverse_polish()では、まず計算に使用するためのStackを生成した。
次に、入力された文字列を「,」で分割し、それぞれの要素を順番に処理した。
要素が演算子（+、−、×、÷）である場合には、Stackから最後の二つの値をpop()によって取り出し、それらをオペランドとして使用した。
その後、switch文によって対応する演算を実行し、計算結果を再びStackへpush()した。
要素が演算子ではない場合、すなわち数値である場合には、その値をdouble型へ変換し、そのままStackへpush()した。
以上の処理を最後まで繰り返すと、Stackには計算結果のみが残る。
そのため、最後にstack.pop()を行い、その値を戻り値として返す実装とした。

回答4.3 :

reverse_polish()から返された計算結果をansとして受け取り、int型へ変換してoutputに表示した。
また、計算終了後にはbuffを空にし、次の入力を受け付けるためにinputも初期化した。
さらに、不正な数式が入力された場合にはExceptionを捕捉し、outputに「Invalid Syntax」を表示するようにした。
